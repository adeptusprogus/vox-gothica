14 — Reference Implementation (informative)

1. Stack

Python 3.11+, Lark for parsing, tomllib/tomli-w for manifests, click for CLI, terraform as subprocess. A Go/Rust rewrite is sanctioned once the conformance suite exists — the suite, not the Python code, is the true reference.

3. Key implementation notes

  • Environments: chained dicts; captured variables promoted to heap cells at closure creation (05-rites §4). Loop variables freshly bound per iteration.
  • Heresy unwinding: Python exceptions carrying the HERESIS object; Major heresies use a distinct exception class that TEMPTA handlers re-raise after running DENIQUE blocks.
  • Deferred references: a Relatio runtime value wrapping a Terraform address path. __getattr__-style access extends the path; every other operation raises relatio_differata_tacta. The emissor serializes it to ${...}.
  • Determinism in FABRICA: the stdlib registry has a per-mode capability mask; fortuna/tempus.nunc*/AUSCULTO are masked in FABRICA (10-fabrica §8).
  • Interpolation: the lexer splits STRING into parts; each expression part is parsed by the same grammar entry expr — no separate mini-language to maintain.
  • Conformance suite: probationes/conformitas/ — every numbered claim in 01–10 gets a passing and a failing case; every heresy genus gets a test asserting its exact genus. CI runs the suite plus gothica proba on all doc examples extracted from 13-examples.md (doc-tested, so docs cannot rot).

5. Deliberate exclusions (v0.2 doctrine)

No concurrency (CHORUS reserved), no FFI (LEGATUS reserved), no pattern matching (MUTATIO reserved), no operator overloading (ever), no inheritance (ever), no macro system (ever — code that writes liturgy is how Chaos gets in), no Terraform CAPITULUM passthrough modules (helper litaniae cover the need in pure Vox Gothica).
