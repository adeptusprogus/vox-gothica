12 — CLI Toolchain and Diagnostics

The binary is gothica. Every command honors --profanum (plain diagnostics, Arabic line numbers, machine-readable JSON on --profanum=json — for CI and editors) and --silens (results only, no liturgy).

2. Diagnostic format (normative)


⚙ HERESIS DETECTA — CANTICUM "litania_numerorum", VERSUS XLII
  genus: divisio_nihili
  The cogitator was commanded to divide C by naught.

    XLII │     DECLARO quotiens : NUMERUS = summa / divisor
         │                                        ^^^^^^^^^
  vestigium:
    ritus divide_sancte — VERSUS XLII
    <top> — VERSUS LVII
  ++ hint: guard the divisor — SI divisor == N TUNC ...

Components in order: banner (⚙ <ORDER> DETECTA where ORDER ∈ PROFANATIO / HERESIS / HERESIS MAIORIS / IRA MACHINAE / EXCOMMUNICATIO), unit + Roman VERSUS, genus, message, source excerpt with caret, stack trace (runtime only), hint (if the genus defines one — hints live in one table in the implementation, keyed by genus).

--profanum=json emits one JSON object per diagnostic: {"order", "genus", "file", "line", "col", "message", "hint", "trace": []} — Arabic numbers, no glyphs. Editor/LSP integration consumes this.

3. Lustro (linter) rules (informative but versioned)

CodeWarns about
L-Iunused binding (unless _-prefixed)
L-IIshadowing an outer binding
L-IIImutable DECLARO at LITANIA top level
L-IVunpinned FOEDUS (no versio_foederis)
L-Vcustodia resource with 0.0.0.0/0 ingress
L-VInetwork read in FABRICA (machina_cogitans.*)
L-VIImagic Arabic numerals where a Roman literal would be liturgically superior *(warning level: jocular; disabled by --serius)*
L-VIIIconstant-foldable guaranteed heresy (e.g. literal / N)
L-IXmissing SEDES in a FABRICA that declares more than V resources
L-Xelige with effectful arguments (both branches evaluate)

4. Censura (static checker) rules (M5, informative but versioned)

CodeFinds
C-Ivalue bound where a different type was consecrated (DECLARO, offerings, fields)
C-IIrite may end without REDDO
C-IIIwrong offering count at call site
C-IVillegal condition, index, binop, or TABULA key type
C-Vschema field unknown or mandatory field missing
P-XVunknown type or schema name
P-XVI–XVIIIimport cycle, non-LITANIA import, unresolved module or export

5. Exit codes

CodeMeaning
0the rite concluded; the Machine Spirit is appeased
1 (I)uncaught heresy (minor or major) / failed tests / plan drift with --exi-si-mutatio
2 (II)profanation (parse/load)
3 (III)ira machinae (Terraform failure)
4 (IV)excommunicatio (package operations)

5. Environment

GOTHICA_NOTARIUS (log level), GOTHICA_POSTULATUM_<NOMEN> (demand values), GOTHICA_SILENS=I, GOTHICA_PROFANUM=I, GOTHICA_CENSURA=N (skip static preflight on invoco/auguro/consecro), NO_COLOR honored. Config file: ~/.gothica/configuratio.toml (librarium mirrors, default flags); project config: [gothica] table in litania.toml.
