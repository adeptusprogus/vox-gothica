01 — Lexical Structure

1. Source files

  • Encoding: UTF-8, no BOM. A BOM is PROFANATIO.
  • Extension: .vg.
  • Line endings: \n or \r\n; the lexer normalizes to \n.
  • Maximum nesting depth of blocks: 256 (deeper is PROFANATIO, code P-VII).

2. Comments

Vox-transmission markers:


++ line comment, to end of line

+++
block comment
may nest: +++ inner +++ is allowed and tracked
+++

Block comments nest (unlike C). An unterminated block comment is PROFANATIO (P-II).

3. Identifiers


ident = [a-z_][a-z0-9_]*
  • Lowercase ASCII only. Keywords are uppercase, so collision is impossible by construction.
  • Convention (informative): snake_case; schema names are nouns (servitor); rite names are verbs or nouns of action (benedictio, divide_sancte).
  • Identifiers beginning with _ are "unhallowed": the lustro linter does not report them as unused.
  • Maximum length: 128 bytes (PROFANATIO P-III beyond).

5. Literals

5.1 Integer literals — NUMERUS

Two interchangeable numeral systems, resolved in the lexer (the parser sees only INT tokens):

  • Arabic: 0, 42, 1_000_000 (underscores as separators, not leading/trailing/doubled).
  • Roman: uppercase, strict subtractive notation: XLII (42), MMXXVI (2026), N (zero — nulla).

Rules for Roman literals (normative):

  1. Symbols: I V X L C D M, values 1/5/10/50/100/500/1000.
  2. Only canonical subtractive forms: IV IX XL XC CD CM. IIII is PROFANATIO (P-V, "Non-canonical numeral. The Machine God abhors sloppiness.").
  3. Range: N to MMMCMXCIX (0–3999). Larger integers must be Arabic. (informative: vinculum notation is a rejected extension — the ambiguity isn't worth it.)
  4. A Roman literal is recognized only where an expression is expected and only if the token is a valid numeral; otherwise the same characters parse as... nothing, since identifiers are lowercase. There is no ambiguity by construction.
  5. Roman literals are non-negative; write -XII (unary minus applied to XII).

5.2 Float literals — FRACTIO

3.14, 0.5, 2.0e10, 1_000.25. A leading or trailing bare dot is PROFANATIO (.5 and 5. are invalid — write 0.5, 5.0). There are no Roman floats; the Omnissiah did not sanction fractional numerals.

5.3 String literals — SCRIPTUM

Double-quoted, single-line by default:


"Ave, Imperator!"
"Line\nbreak, tab\t, quote\", backslash\\, unicode\u{2620}"

Escapes: \n \t \r \" \\ \0 \u{XXXXXX}. Unknown escapes are PROFANATIO (P-VI).

Interpolation. {expr} embeds any expression; {{ and }} are literal braces:


"Servitor {nomen} holds rank {gradus + I}"

Interpolated expressions are parsed by the main grammar (a string literal token is split by the lexer into text parts and expression parts). The value is converted with the same rules as scriptum(x) (§02-types, §8).

Multi-line strings ("great chants"): triple-quoted, leading indentation stripped to the closing delimiter's column:


DECLARO litania : SCRIPTUM = """
    Praise the Omnissiah,
    from whom all knowledge flows.
    """

5.4 Boolean and null

VERUM, FALSUM (type VERITAS); NIHIL (type NIHIL).

5.5 Collection literals


[I, II, III]                          ++ ORDO
{ "terra": M, "mars": D }             ++ TABULA
servitor { nomen: "Kane", gradus: VII }   ++ SCHEMA instance (see 02-types)

Trailing commas are permitted in all bracketed literals.

6. File header


AVE OMNISSIAH.                 ++ salutation: optional, at most once, must be first
CANTICUM "salutatio".          ++ or FABRICA "..." or LITANIA "..."
  • The header statement is mandatory and must precede all other statements. A file without one is PROFANATIO (P-I, "This scroll bears no seal.").
  • Header statements are the only statements terminated by . — a deliberate liturgical flourish and a cheap way to make the header visually distinct.
  • The header name must match [a-z_][a-z0-9_]* inside the quotes; it names the unit in logs, diagnostics, and (for FABRICA) the Terraform workspace directory.

7. Statement termination and line continuation

  • Statements terminate at newline.
  • A newline does not terminate a statement when it occurs: inside unclosed (, [, {; immediately after a binary operator (+ - * / % == != < <= > >= ET VEL), after FIAT, after a comma, or after ->.
  • There is no explicit statement separator; two statements on one line are PROFANATIO (P-VIII). One line, one utterance.

8. Significant tokens summary (informative)

INT FLOAT STRING_PARTS IDENT + keywords + operators + - * / % == != < <= > >= ( ) [ ] { } , : . -> + NEWLINE. Indentation is not significant; blocks are delimited by keywords (... FINIS X). The purga formatter enforces four-space indentation as canon, but the parser ignores it.
