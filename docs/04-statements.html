04 — Statements and Control Flow

1. Declarations


DECLARO nomen : NUMERUS = XII      ++ mutable vessel
SANCTUM pi : FRACTIO = 3.14159     ++ consecrated: immutable forever
  • Initializer mandatory. Declared type checked against the value (typus_profanus at runtime if violated).
  • Reassigning a SANCTUM raises sanctum_violatum. Note: SANCTUM consecrates the binding, not the object — a SANCTUM ordo can still be mutated through adde. For deep immutability use sigilla(x) (freezes an ORDO/TABULA/SCHEMA recursively; subsequent mutation raises sigillum_fractum).
  • Redeclaration in the same scope: PROFANATIO (P-XI). Shadowing an outer scope: legal, but lustro warns.

2. Assignment


lvalue FIAT expr

FIAT ("let it become") is the sole assignment form. No compound assignment, no increment — write the litany in full: i FIAT i + I.

3. Scope (normative)

  • Lexical scoping. Scopes: module (file) → rite body → each block (SI arm, loop body, TEMPTA arms).
  • A name is visible from its declaration to the end of its scope. Use before declaration: vas_ignotum.
  • Loop variables (PRO OMNI x) are scoped to the loop body and are immutable within it (assignment → sanctum_violatum). Each iteration binds a fresh x (this matters for closures — see 05-rites §4).
  • Top-level code in a module runs exactly once, at import/invocation (07-modules).

4. Conditional


SI vigor > C TUNC
    VOCIFERO "Strong."
ALITER SI vigor > L TUNC
    VOCIFERO "Willing."
ALITER
    VOCIFERO "Appease it!"
FINIS SI

Condition must be VERITAS (no truthiness — 02-types §10). Any number of ALITER SI arms; at most one final ALITER.

5. Loops


DUM cond AGE ... FINIS DUM                 ++ while

PRO OMNI x IN iterable AGE ... FINIS PRO   ++ for-each

Iterables: ORDO (elements, in order), TABULA (keys, insertion order), SCRIPTUM (code points as 1-char SCRIPTUM), series(a, b) / series_gradu(a, b, gradus) (integer ranges, half-open [a, b); gradus of N raises argumentum_pravum; negative gradus counts down).

  • Mutating an ORDO/TABULA while iterating it raises mutatio_in_processione ("mutation during the procession") at the moment of the next iteration step. (Deterministic and immediate — no C++-style undefined behavior.)
  • RUMPO breaks the innermost loop; PERGO skips to the next iteration. Either outside a loop: PROFANATIO (P-XII). There are no labeled breaks; refactor into a rite and REDDO.
  • There is no C-style for; series covers it.
  • Infinite loop idiom: DUM VERUM AGE ... FINIS DUM — sanctioned; the linter stays silent if the body contains RUMPO or REDDO.

6. Output and input


VOCIFERO expr      ++ stdout + newline; value converted via canonical scriptum
SUSURRO expr       ++ stderr + newline
DECLARO s : SCRIPTUM = AUSCULTO()    ++ read one line from stdin, newline stripped

AUSCULTO() at EOF raises auscultatio_vana ("the listening was in vain") — catchable, so interactive loops can end gracefully.

8. Entry point

  • A CANTICUM executes top-level statements in order. If a rite named principium ("beginning") is declared, it is called (with no arguments) after top-level statements complete; its declared return type must be NIHIL or NUMERUS (a NUMERUS becomes the process exit code).
  • imperium.exi(codex) exits immediately with the given code.
