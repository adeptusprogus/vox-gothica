02 — Types and Values

Vox Gothica is statically declared, dynamically checked in v0.2: every binding carries a type annotation; the interpreter enforces it at every assignment, argument pass, return, and index. A full static checker is milestone M5 (14-implementation.md); the annotations already make it possible without new syntax.

2. Value vs reference semantics (normative)

  • Value types (NUMERUS FRACTIO SCRIPTUM VERITAS NIHIL) are immutable. Assignment copies.
  • Reference types (ORDO TABULA schema instances RITUS HERESIS) are handles. Assignment aliases:

DECLARO a : ORDO[NUMERUS] = [I, II]
DECLARO b : ORDO[NUMERUS] = a
adde(b, III)
VOCIFERO longitudo(a)     ++ III — a and b are one vessel

To copy: exemplar(x) performs a deep copy of ORDO/TABULA/SCHEMA (rite values are not copyable — heresy conversio_impossibilis).

3. NUMERUS semantics

  • Range: −2⁶³ … 2⁶³−1. Overflow raises heresy excessus_numeri — it does not wrap. (Rationale: silent wrapping is precisely the kind of quiet corruption the Mechanicus fears most.)
  • / on two NUMERUS is integer division, truncating toward zero. % takes the sign of the dividend. x / 0 and x % 0 raise divisio_nihili.
  • Mixed NUMERUS op FRACTIO promotes to FRACTIO — the only implicit conversion in the language.

4. FRACTIO semantics

IEEE-754: NaN, +inf, -inf exist (constants mathematica.NAN, mathematica.INFINITUM). 0.0 / 0.0 is NaN, not a heresy — floats follow the IEEE creed, which predates the Imperium and is respected. NaN != NaN is VERUM. fractio_integra(x) raises conversio_impossibilis on NaN/inf.

5. SCRIPTUM semantics

  • Immutable sequence of Unicode code points. longitudo counts code points; scriptura.mensura_octetorum counts UTF-8 bytes.
  • Indexing s[i] returns a one-code-point SCRIPTUM; out of range raises index_extra_fines. Negative indices are heresy (index_extra_fines) — there is no Pythonic -1; use s[longitudo(s) - I]. Explicit is pious.
  • Slicing: pars(s, a, b) returns [a, b); clamps to bounds (does not raise).
  • + concatenates two SCRIPTUM. SCRIPTUM + NUMERUS is typus_profanus — interpolate or convert.

6. SCHEMA — records


SCHEMA servitor
    nomen : SCRIPTUM
    gradus : NUMERUS = I            ++ field default
    officia : ORDO[SCRIPTUM] = []
FINIS SCHEMA
  • Declares a new named type, usable wherever a type is expected: DECLARO s : servitor = ..., ORDO[servitor], parameter and return types.
  • Instantiation uses the schema name as constructor:

DECLARO s : servitor = servitor { nomen: "Kane", gradus: VII }

Fields without defaults are mandatory at instantiation; omitting one raises campus_deest. Unknown fields raise campus_ignotus.

  • Field access and mutation: s.gradus, s.gradus FIAT VIII. Fields are typed; assignment checks types.
  • Instances are reference values; exemplar(s) deep-copies; == compares structurally (same schema, all fields equal).
  • SCHEMA declarations are allowed only at top level (of any mode). No methods in v0.2 — write rites that take the schema as first parameter. Methods (RITUS ... APUD servitor) are a v0.3 candidate.
  • No inheritance. Ever. (This is doctrine, not a roadmap gap: composition is purer.)
  • Recursive schemas are permitted if the recursion passes through ORDO/TABULA (which may be empty) or a NIHIL-permitting field, declared as pater : servitor = NIHIL — a field whose default is NIHIL is nullable; all other fields never hold NIHIL.

7. NIHIL discipline (normative)

  • NIHIL may be assigned only to: variables declared with type NIHIL (rarely useful), nullable schema fields (§6), and returned from -> NIHIL rites.
  • Passing NIHIL where any other type is expected raises typus_profanus.
  • Any operation on NIHIL (arithmetic, indexing, field access, calling) raises nihil_tactum.
  • Test for it: x == NIHIL is the only legal operation, allowed on nullable fields.

(Rationale: NIHIL is quarantined, not banned — Terraform attributes and JSON need it, but it must never wander the program freely.)

8. Conversions (all explicit)

scriptum canonical forms (normative, used by interpolation and VOCIFERO): integers in Arabic; floats in shortest round-trip form; VERUM/FALSUM; NIHIL; lists as [a, b]; maps as {k: v, ...} in insertion order; schema instances as nomen{campus: valor, ...}; rites as RITUS <nomen>.

9. Equality (normative)

  • ==/!= is structural, deep equality for all types. NUMERUS 42 ≠ FRACTIO 42.0 (== across distinct types is simply FALSUM, not a heresy — comparison is a question, and questions are not heresy).
  • Identity of references: built-in idem(a, b) -> VERITAS.
  • Rite values: == is identity.
  • Ordering < <= > >= is defined for NUMERUS↔NUMERUS, FRACTIO↔FRACTIO, mixed numeric (promotes), and SCRIPTUM↔SCRIPTUM (lexicographic by code point). Anything else: typus_profanus.

10. No truthiness (normative)

Conditions (SI, DUM) require VERITAS. SI longitudo(x) TUNC is typus_profanus. Write SI longitudo(x) > N TUNC. Ambiguity is the door through which heresy enters.
