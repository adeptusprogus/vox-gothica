  • Comparison chains are PROFANATIO (P-IX): a < b < c must be a < b ET b < c. (One litany, one claim.)
  • No ternary operator; use SI/ALITER or the built-in elige(cond, a, b) — note elige evaluates both branches (it is an ordinary rite), which the docs warn about.
  • No assignment expressions. FIAT is a statement.
  • No user-defined operator overloading. The Machine God fixed the operators at the dawn of the age.

2. Evaluation order (normative)

Strict left-to-right, everywhere: operands, call arguments, list/map literal elements, interpolation segments. ET/VEL short-circuit (right operand not evaluated when the left decides). Both operands of all other binary operators are always evaluated.

4. Indexing and fields

  • ordo[i]: i must be NUMERUS, 0 <= i < longitudo — else index_extra_fines. Index expressions of any other type: typus_profanus.
  • tabula[k]: missing key raises clavis_ignota. Use habet(t, k) first, or accipe(t, k, defectus) for a default.
  • scriptum[i]: one-code-point SCRIPTUM (02-types §5).
  • instantia.campus: unknown field → campus_ignotus.
  • Assignment targets: x, ordo[i], tabula[k] (inserts if absent), instantia.campus, and chains thereof (a[i].b[k] FIAT v). Assigning through anything else is PROFANATIO (P-X).

5. Calls

  • Arity is exact: wrong count → argumentum_pravum. No varargs, no defaults, no keyword arguments in v0.2 (rejected for now: each adds parser and doc complexity that the language does not yet earn back).
  • Arguments are passed by value for value types, by reference for reference types (02-types §2).

6. String interpolation

"{expr}" evaluates the expression and converts via canonical scriptum (02-types §8). Interpolations may nest arbitrarily ("{elige(x > N, "plus", "minus")}"). A deferred infrastructure reference (10-fabrica §7) inside interpolation is legal only in FABRICA attribute/PROFITEOR contexts and compiles to a Terraform template string.

7. Constant expressions (informative)

The lustro linter folds constant expressions to warn about e.g. guaranteed divisio_nihili. The interpreter itself performs no folding in v0.2 — behavior is identical either way because the language is deterministic and effect-ordered.
