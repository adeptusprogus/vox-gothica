05 — Rites (Functions)

1. Declaration


RITUS benedictio (nomen : SCRIPTUM, gradus : NUMERUS) -> SCRIPTUM
    SI gradus > V TUNC
        REDDO "Magos {nomen}"
    FINIS SI
    REDDO "Adept {nomen}"
FINIS RITUS
  • Parameter types and return type are mandatory. -> NIHIL rites may omit REDDO or use bare REDDO.
  • A non-NIHIL rite whose execution reaches FINIS RITUS without REDDO raises reditus_deest ("the return is missing") at that moment. (Runtime in v0.2; the M5 static checker will make it compile-time.)
  • Returned value is type-checked against the declared return type (typus_profanus).
  • Rite declarations are allowed at top level and inside other rites (local rites). Not inside blocks (SI/loops) — PROFANATIO (P-XIII).
  • No overloading, no varargs, no default parameter values, no keyword arguments (03-expressions §5).

2. Rites as values

Rites are first-class:


DECLARO f : RITUS = benedictio
VOCIFERO f("Kane", VII)

RITUS applica (opus : RITUS, x : NUMERUS) -> NUMERUS
    REDDO opus(x)
FINIS RITUS

The RITUS type is opaque in v0.2: any rite value fits any RITUS slot; arity/type errors surface at call time (argumentum_pravum / typus_profanus). Parameterized rite types RITUS(NUMERUS) -> NUMERUS are reserved syntax for M5.

3. Anonymous rites

A rite expression without a name:


DECLARO duplica : RITUS = RITUS (x : NUMERUS) -> NUMERUS
    REDDO x * II
FINIS RITUS

DECLARO quadrata : ORDO[NUMERUS] =
    transmuta(series(I, VI), RITUS (x : NUMERUS) -> NUMERUS
        REDDO x * x
    FINIS RITUS)

Anonymous rites are ordinary expressions (grammar: rite_expr) and may appear anywhere an expression may. There is no one-line lambda sugar in v0.2 (considered and deferred: a second function syntax doubles every tutorial).

4. Closures (normative)

  • A rite captures its lexical environment by reference: it sees later mutations of captured DECLARO variables.
  • Loop variables are rebound each iteration (04-statements §3), so the classic bug is defined away:

DECLARO opera : ORDO[RITUS] = []
PRO OMNI i IN series(N, III) AGE
    adde(opera, RITUS () -> NUMERUS
        REDDO i          ++ captures THIS iteration's i
    FINIS RITUS)
FINIS PRO
++ opera[N]() == N, opera[I]() == I, opera[II]() == II
  • Captured variables live as long as any capturing rite (the implementation must heap-allocate captured cells).

5. Recursion

  • Direct and mutual recursion are permitted. Forward references work at top level: all top-level rite names in a module are hoisted before top-level code runs.
  • Default recursion depth: MMMM (4000) frames; exceeding it raises profunditas_excessa (catchable). Configurable: gothica invoco --profunditas 10000.
  • No tail-call elimination in v0.2 (documented so programs don't depend on it accidentally).

6. Purity annotation (informative)

SANCTUM RITUS nomen (...) declares intent of purity; lustro warns if the body performs I/O or mutates captured state. The interpreter does not enforce it in v0.2. Pure rites are candidates for consecration-time constant folding in FABRICA mode.
