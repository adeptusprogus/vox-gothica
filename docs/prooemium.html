Vox Gothica

A High Gothic programming language for canticles and infrastructure.

Version 0.2.0-draft ("Second Canticle") · File extension .vg · Docs are normative unless marked (informative).

"The soul of the Machine God surrounds thee. The power of the Machine God invests thee. Live!"

Vox Gothica is a language whose source reads as the pseudo-Latin liturgy of the Adeptus Mechanicus. One syntax, two purposes:

  1. Canticles — general-purpose programs: an interpreted, imperative, lexically-scoped language with rites (functions), schemas (records), heresy handling (errors), modules, and packages ("litaniae").
  2. Fabricae — infrastructure manifests that transpile to Terraform JSON. Terraform keeps state, providers, and diffing; Vox Gothica is the liturgical frontend.

Design principles (normative)

  1. Pseudo-Latin, fixed forms. Every keyword is one Latin word frozen in one grammatical form. The parser never conjugates. (One sanctioned exception: HERESIS/HERESIM, nominative/accusative — see §15.)
  2. Code is liturgy. Programs are canticles, functions are rites, packages are litaniae, errors are heresies, deployment is consecration. The metaphor is total: it covers the CLI, diagnostics, the package registry, and the testing framework.
  3. Serious core, ironic surface. Underneath: a small, honest, fully implementable language. The joke never costs correctness, debuggability, or Terraform interoperability.
  4. One AST, two backends. Imperative code executes; infrastructure declarations accumulate into a resource graph emitted as .tf.json.
  5. No lock-in. The Terraform escape hatch is a guarantee: generated JSON is readable, state is ordinary Terraform state, and --profanum mode strips all theming for CI.
  6. Strictness is piety. No implicit coercions, no truthiness, mandatory type annotations, immutable-by-keyword constants. Sloppiness is how heresy enters.

Conformance

An implementation conforms if it: accepts every program this spec declares valid; raises exactly the heresy genus specified in 06-heresies.md for every condition listed there; produces Terraform JSON per 10-fabrica.md; and passes the conformance suite (probationes/ in the reference repo, milestone M4).

Undefined behavior does not exist in Vox Gothica: anything not defined here is PROFANATIO (a compile-time rejection). When this spec is ambiguous, the stricter reading is correct — that is doctrine.
