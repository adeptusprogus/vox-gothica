10 — Fabrica (Infrastructure)

Model: a FABRICA file is a program executed once at consecration time; its side effect is a resource graph; the graph is emitted as Terraform JSON (.tf.json); terraform owns state, providers, planning, and drift. Vox Gothica adds no second source of truth — that guarantee is what makes it safe to use ironically in production.

1. Consecration pipeline


fabrica.vg ──interpret──▶ resource graph ──emit──▶ .gothica/<name>/main.tf.json
                                                        │
                              terraform init/plan/apply ◀┘   (subprocess)
  • Output directory .gothica/<fabrica-name>/ also holds Terraform's .terraform/ and plan files. Add .gothica/ to VCS ignore except *.tf.json if you want reviewable artifacts (lustro suggests committing them — plans should be diffable in PRs).
  • Everything imperative (loops, rites, litania calls) happens before Terraform sees anything. Terraform receives only static JSON.

2. Covenants (providers)


FOEDUS "aws"
    regio: "eu-west-1"
    versio_foederis: "~> 5.0"        ++ provider version constraint
FINIS FOEDUS

Maps to provider + terraform.required_providers. Multiple covenants with the same provider need cognomen: (alias). versio_foederis is extracted into required_providers; omitted = unpinned (with a lustro warning — unpinned covenants are reckless piety).

3. The Seat (state backend)


SEDES "s3"
    bucket: "mechanicus-status"
    key: "fabricae/interretialis.tfstate"
    regio: "eu-west-1"
FINIS SEDES

Maps to terraform.backend. At most one per FABRICA (nomen_geminum otherwise). Absent = local state, and auguro prints a gentle admonition: "The state rests upon this fragile machine alone."

4. Raising resources


EXSTRUATUR machina "web_primus"
    genus: "aws_instance"
    imago: scrutinium.ubuntu.id
    genus_machinae: "t3.micro"
    tags:
        Name: "web-primus"
    FINIS
FINIS EXSTRUATUR
  • The word after EXSTRUATUR is the species — a required human category: machina (compute), retiaculum (network), horreum (storage), custodia (security/IAM), nuntius (queues/messaging), codex (DNS/config), res (anything else). Species is metadata: it drives lint rules (a custodia with open 0.0.0.0/0 ingress gets a louder warning) and reference readability. The Terraform type comes from mandatory genus:.
  • Resource name: string, may interpolate consecration-time values; must evaluate to [a-z_][a-z0-9_]* (else argumentum_pravum) and be unique across the FABRICA (nomen_geminum, Major).
  • Attribute values: full expressions, evaluated at consecration; nested blocks end with bare FINIS; lists of blocks are lists of TABULA literals.
  • A NIHIL attribute value means "omit this attribute" — the sanctioned use of NIHIL (02-types §7).

6. Demands, proclamations, divinations


POSTULO ambitus_nomen : SCRIPTUM                     ++ variable, required
POSTULO numerus_servorum : NUMERUS = III             ++ with default
POSTULO clavis_arcana : SCRIPTUM ARCANUM             ++ sensitive: masked in output

PROFITEOR primus_ip = machina.web_primus.public_ip   ++ output
PROFITEOR arcanum_db = res.db.password ARCANUM       ++ sensitive output

SCRUTOR imago_recens "ubuntu"                        ++ data source
    genus: "aws_ami"
    most_recent: VERUM
FINIS SCRUTOR
  • POSTULO values are resolved before interpretation (precedence: -postulatum k=v flags → postulata.toml → env GOTHICA_POSTULATUM_<NOMEN> → default → else abort with IRA MACHINAE-adjacent prompt listing missing demands). They are ordinary typed values usable in any imperative logic — unlike Terraform's lazy variables. They are also emitted as variable blocks (with values pinned via generated .tfvars.json) so the JSON stands alone.
  • ARCANUM marks sensitive: masked in gothica output, sensitive = true in Terraform.

7. Deferred references (normative)

<species>.<nomen>.<attr...> and scrutinium.<nomen>.<attr...> evaluate to deferred references — values that exist only after terraform apply.

  • Legal positions: resource/covenant attribute values, PROFITEOR expressions, and inside string interpolation in those positions (compiles to "${...}" template strings).
  • Any other use — a SI condition, arithmetic, longitudo(), storing then branching — raises relatio_differata_tacta (Major): "Thou hast touched that which is not yet made flesh." The interpreter enforces this by making deferred references a distinct runtime kind that poisons every operation except attribute access (which extends the reference) and placement into infra positions.
  • Cross-referencing implies ordering; explicit ordering uses postulat: [machina.db, res.certificate].

8. Generating resources imperatively


PRO OMNI i IN series(N, numerus_servorum) AGE
    EXSTRUATUR machina "serv_{i}"
        genus: "aws_instance"
        imago: scrutinium.ubuntu.id
        genus_machinae: elige(i == N, "t3.medium", "t3.micro")
    FINIS EXSTRUATUR
FINIS PRO

Runs at consecration; registers serv_0…. Determinism rule (normative): a FABRICA must be a pure function of (source, POSTULO values, litaniae versions). fortuna.*, tempus.nunc*, AUSCULTO raise fabrica_impura (Minor) in FABRICA mode; archivum.lege and machina_cogitans.pete are permitted (reading configuration is legitimate) but lustro flags network reads. *(Without this rule, auguro and consecro could see different worlds — the plan would be a lie.)*

9. Reusable infrastructure: FABRICA helper litaniae

The Terraform-module equivalent is plain code (08-litaniae §9):


++ litania github.com/adeptus/vpc_benedicta, module fons/vpc.vg
AVE OMNISSIAH.
LITANIA "vpc".

RITUS vpc_benedicta (nomen : SCRIPTUM, cidr : SCRIPTUM) -> NIHIL
    EXSTRUATUR retiaculum "{nomen}_vpc"
        genus: "aws_vpc"
        cidr_block: cidr
    FINIS EXSTRUATUR
    ++ ... subnets, route tables, the whole blessed pattern
FINIS RITUS

Called from any FABRICA: vpc.vpc_benedicta("fabrica_prima", "10.0.0.0/16"). Called from a CANTICUM: fabrica_in_cantico. Rites may return deferred references to let callers wire outputs together — a rite's return type for these is declared SCRIPTUM and the deferral kind rides along at runtime *(v0.2 pragmatism; a RELATIO type is an M5 candidate)*.

11. Lifecycle commands

auguro (plan), consecro (apply, confirmation word FIAT), exterminatus (destroy, confirmation word EXTERMINATUS, no bypass flag), auguro --exi-si-mutatio (exit I if plan non-empty — for CI drift detection). All accept --profanum for untheming. Terraform failures: exit III, banner + verbatim Terraform stderr (06-heresies §9).
